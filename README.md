# 学习笔记
我先从Git官网下载了适合我电脑系统的安装包，按照安装向导一步步完成了安装。安装完成后，我在Git Bash中使用命令git config --global user.name "用户名"和git config --global user.email "邮箱"配置了我的用户名和邮箱，这样Git就能知道我是谁，在提交代码时可以对我进行标识。
我选择了一个合适的文件夹作为我的项目目录，在该文件夹中打开Git Bash，执行git init命令初始化了一个新的本地仓库。这个命令会在文件夹中创建一个隐藏的.git子目录，里面包含了Git仓库的所有必要文件。 
在关联远程仓库时，我遇到了权限不足的问题，无法将本地仓库与远程仓库关联起来。后来我了解到需要生成SSH密钥对。我使用命令ssh-keygen -t rsa -C "我的邮箱"生成了密钥对，将生成的公钥复制并添加到GitHub的SSH keys设置中，问题就解决了。
在实践过程中，我对Git的命令不是很熟悉，导致一些操作失误。比如，我一开始不太理解git add和git commit的区别和联系。通过查阅学习资料，我逐渐熟悉了这些命令的用法，明白了它们在Git工作流程中的作用。
通过本次实践，我对Git有了更深入的了解，也体会到了它作为代码版本管理工具的强大之处。Git能够有效地管理代码的版本变化，方便我在不同的设备上进行开发工作，同时也为团队协作提供了便利。在实践中，我学会了如何安装配置Git、创建本地和远程仓库、关联本地与远程仓库以及推送代码等基本操作。
